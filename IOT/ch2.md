### 아두이노

---

- 목적
    
    마이크로컨트롤러에서 실행되는 프로그램을 개발하는 것
    
- 아두이노 보드
    1. 마이크로 컨트롤러 : 컴퓨터 본체에 해당하는 기능을 하나의 칩으로 집약해 놓은 앉은 성능의 컴퓨터
    2. 마이크로 컨트롤러 보드 : 마이크로 컨트롤러 이외에 핀 헤더, USB 커넥터 및 2개의 ICSP 핀 헤더, 전원 공급을 위한 커넥터가 포함된 컴퓨터

| 아두이노 | 마이크로컨트롤러 | 아키텍처 | 클록 | CPU 비트 |
| --- | --- | --- | --- | --- |
| 우노 | ATmega328 | AVR | 16MHz | 8비트 |
| 나노 | ATmega328 | AVR | 16MHz | 8비트 |
| 레오나르도 | ATmega32u4 | AVR | 16MHz | 8비트 |
| 메가2560 | ATmega2560 | AVR | 16MHZ | 8비트 |

아두이노 우노 : ATmega328 마이크로 컨트롤러를 사용하는 기본 보드

아두이노 레오나르도 : USB연결을 지원하는 ATmega32u4 마이크로 컨트롤러를 사용함으로써 마우스, 키보드 등과 같은 USB장치를 쉽게 만들 수 있음

아두이노 메가2560 : 아두이노 우노보다 3배 많은 입출력 핀과 8배 큰 프로그램 메모리를 제공하기에 복잡한 프로그램을 만들기 좋음.

- 마이크로 컨트롤러
    - 마이크로 프로세서 : 컴퓨터의 중앙처리장치를 하나의 IC(Integrated Circuit)칩으로 집적하여 만든 반도체 소자
        - 장점 : 제품의 소형화 및 경량화, 저렴한 가격, 융통성
        - 단점 : 처리능력, 범용성
    - 마이크로 컨트롤러 : 하나의 칩에 중앙처리장치, 일정 용량의 메모리와 입출력 인터페이스까지 내장한 특수 목적용 마이크로 프로세서
        - 필요성 : 허드웨어로 수행할 작업 일부를 소프트웨어로 대체 가능함

- CISC & RISC
    - CISC(Complex Instruction Set Computer) : 모든 고급언어 문장들에 대해 각각 기계 명령어가 대응되도록 하는 것. 흔히 사용되는 인텔의 CPU는 CISC구조를 주로 사용.
    - RISC(Reduced Instruction Set Computer) : CISC의 많은 명령어 중 주로 쓰이는 것만을 추려서 하드웨어로 구현하는 것. 아두이노 보드에 사용된 마이크로 컨트롤러의 CPU는 대부분 RISC 구조를 사용

|  | CISC | RISC |
| --- | --- | --- |
| 명령어의 개수 | 많음 | 적음 |
| 프로그램 크기 | 작음 | 큼 |
| 하드웨어 복잡도 | 높음 | 낮음 |
| 소프트웨어(컴파일러) 복잡도 | 낮음 | 높음 |
| 명령당 클록 수 | 가변 | 고정 |
| 전력 소모 | 많음 | 적음 |
| 호환성 | 높음 | 낮음 |
| 대표적인 CPU 회사 | 인텔 | ARM |

- 폰노이만 구조와 하버드 구조
    - 폰노이만 구조 : 데이터와 명령어가 한 곳에 저장. 일반적으로 DRAM을 사용. 마이크로 프로세서는 폰 노이만 구조 채택
    - 하버드 구조 : 명령어와 메모리가 별개의 저장소에 저장. 프로그램 메모리로 플래시 메모리를, 데이터 메모리로 SRAM 사용. 마이크로 컨트롤러는 하버드 구조 채택

| 항목 | 마이크로 컨트롤러 | 데스크톱 컴퓨터 |
| --- | --- | --- |
| CPU | ATmega328 | Intel Core i7 |
| 비트 | 8 | 64 |
| 주기억장치 | 2KB(SRAM) | 16GB(DRAM) |
| 보조기억장치 | 32KB(플래시 메모리) | 512GB(SSD) |
| 클록 | 16MHz(단일 코어) | 1.8GHz(4 코어) |

- 프로그램이 만들어지는 과정
    - ATmega328 마이크로 컨트롤러의 플래시 메모리 용량은 32kb에 불과
    - 따라서 데스크톱 컴퓨터에서 개발하고, 실행 파일만을 마이크로 컨트롤러로 업로드하여 설치하는데 이를 교차개발환경이라 한다.
    - AVR시리즈 마이크로 컨트롤러에서 프로그램 업로드 방법
        - 시리얼 방식 : UART 시리얼 통신
        - ISP(In System Programming) 방식 : SPI(Serial Peripheral Interface) 통신
    - 과정
        1. AVR 툴체인에 포함된 교차 컴파일러를 사용하여 기계어 파일을 생성함
        2. 시리얼 통신을 사용하여 기계어 파일을 전송함

- 마이크로 컨트롤러의 메모리
    - AVR 시리즈 : SRAM(휘발성), 플래시 메모리(비휘발성), EEPROM(비휘발성, 속도 느림) 사용
    - 마이크로 컨트롤러에서 변숫값이 저장되는 곳은 데이터 메모리(SRAM, 휘발성)
    - 아두이노 우노는 데이터 메모리와 프로그램 메모리가 모두 적어, 간단한 제어 장치를 구현하는데 사용할 수 있음
    
    - Cortex - m 시리즈 : SRAM과 플래시 메모리 사용
    - Cortex - m 시리즈 마이크로 컨트롤러는 동작 속도가 빠르고, 데이터 메모리인 SRAM의 크기가 커서 복잡한 계산을 빨리 수행할 필요가 있을 때 선택
    
- ROM과 RAM
    - ROM(Read Only Memory) : 비휘발성 메모리, 쓸 수 있는 횟수에 제한
        - EEPROM은 한 번 내용을 기록하고 자주 참조하는 값을 기록하는 용도.
        - 플래시 메모리는 EEPROM의 변형. 블록 단위의 쓰기를 실행하므로 불필요한 쓰기가 발생할 수 있어 EEPROM에 비해 쓰기 가능한 횟수가 적다.
        - SD카드는 플래시 메모리를 사용하여 만들어진 외부 저장장치 표준의 하나
    
    - RAM(Random Access Memory) : 휘발성 메모리, 읽고 쓰는데 제한이 없고 읽고 쓰는 속도가 빠름
        - 정적 RAM(Static RAM) : 플립플롭을 바탕으로 만들어진 메모리. 한 번 기록된 데이터는 전원이 공급되는 동안 계속 남아있음. 집적도가 낮고 CPU 내의 레지스터나 캐시 메모리 등 작은 크기의 고속 메모리에 사용.
        - 동적 RAM(Dynamic RAM) : 커패시터에 전하를 저장하는 방식으로 데이터를 기록. 전원이 공급되는 동안에도 일정 시간이 지나면 방전으로 인해 데이터가 사라짐. 구조가 간단하여 집적도를 높이기 쉽고 소비 전력이 적어 데스크톱 컴퓨터의 메인 메모리로 사용

- 메모리 구조 정리
- AVR 시리즈 마이크로 컨트롤러에 포함된 메모리
    - 플래시 메모리(프로그램 메모리)
    - SRAM(데이터 메모리)
    - EEPROM(프로그램 실행 중에 임의로 값을 읽고 쓰는 용도)
